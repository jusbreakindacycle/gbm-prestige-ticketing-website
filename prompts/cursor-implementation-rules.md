# Cursor Implementation Rules

Before editing or creating files, always check the approved documents inside the docs folder.

Do not invent features.

Do not create a backend.

Do not create a database.

Do not add payment, booking, login, or admin features.

When asked to build a page:
1. Read the relevant documentation first.
2. Explain the files you will edit.
3. Make small, reviewable changes.
4. Keep the design consistent.
5. Avoid placeholder lorem ipsum.
6. Use real approved project content where available.
7. Mark missing owner-confirmed information clearly.
